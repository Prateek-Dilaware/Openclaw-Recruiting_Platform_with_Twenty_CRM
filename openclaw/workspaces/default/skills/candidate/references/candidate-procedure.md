# Candidate Procedure

## Search

Use metadata discovery before relying on a custom field. Search with the plugin's record list/get tools. Stable identifiers outrank generated similarity. The plugin's people dedup/find-similar tools may help only when the recruiting candidate object maps to People; do not assume that mapping.

## Create and update

Candidate creation and profile updates are informational CRM mutations. Record source/provenance where the current schema supports it. Do not set application, interview, offer, or requisition lifecycle fields from this skill.

## Merge review

Twenty's installed plugin exposes deduplication as read-only. Keep this behavior: produce a merge plan with exact records, fields to retain, relationship conflicts, and audit notes. A human-approved deterministic merge operation is required before any merge executes.

## Timeline

Read candidate relations at supported depth and related application/interview/evaluation records. Report chronological facts; mark missing time data rather than inventing event order.
